SSRF-内网Redis Getshell

Posted on 2020-04-17 In SSRF

Redis是常见的内网服务，因其默认配置未授权访问或使用弱口令认证，可以导致getshell。经过总结，目前网上对于Redis常见的getshell方式有以下几种:

  • 绝对路径写Webshell
  • 写入ssh公钥
  • 写crontab计划任务反弹shell(针对CentOS)
  • 主从复制(4.0 < Redis < 5.0.5)

接下来依次介绍几种方法的细节以及用上的一些脚本，脚本以及此次实验的环境都放在了github上。

0x00 环境配置

  • centos7.0 + Redis5.0
  • Python3.6.0

由于具有SSRF的php代码放置在docker启动的web服务器上，Redis在另一个镜像中，将Redis绑定在0.0.0.0。因此，在宿主机访问curl -v dict://127.0.0.1:6379/info可以成功，但对于docker里的web服务器，docker.for.mac.host.internal为能访问宿主机的地址。

  3. 知道web目录物理路径
